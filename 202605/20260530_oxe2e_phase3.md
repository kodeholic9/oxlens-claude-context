## §1 컨텍스트

- Phase 2로 floor 라우팅 **양성**(granted 봇 audio가 vssrc로 fan-out)은 입증. 단 **gating 음성**(비발화 봇 송신이 *안 새는가*)은 미검증 — 누적 카운터로는 "특정 구간에 안 왔다"를 못 잡는다.
- Phase 3 = **subscriber 수신 RTP에 시각을 찍고, floor 이벤트로 구간을 잘라**, 각 구간에서 *granted 봇 vssrc만 도착*했는지 검산.
- 핵심 난점 = **PTT 전환 구간의 정당한 잔류**(silence flush / keyframe 대기)를 누수로 오판하지 않기 → guard band.

---

## §2 결정된 사항

1. **gating 음성 검증 = floor 이벤트 기반 구간 검산.** floor Granted/Idle 수신 시각으로 phase 경계를 긋고, 각 granted 구간에서 그 봇 vssrc만 도착(양성) + 타 봇 송신 미도착(음성)을 본다.
2. **봇 subscriber 수신 기록을 시각 기반으로 확장** — per-vssrc arrival 시각(또는 구간 버킷 카운트). 누적 총량 카운터에 시각 축 추가.
3. **guard band로 전환 회색지대 제외** — floor 전환 직후 일정 구간(silence flush·keyframe 대기를 덮는 길이)은 음성 판정에서 제외. 길이는 `config.rs` 상수(매직넘버 금지), 서버 PTT 특성 참고해 김과장이 1차 설정 후 보고.
4. **ptt_rapid 시나리오 불변** — 판정 인프라만 추가. 시나리오 TOML 안 건드림.
5. **서버 0 변경** — 시각은 봇 측에서 찍는다(텔레메트리≠검증, 봇 관측). 서버에 검증 카운터 추가 금지.
6. **admin 삼각검증은 Phase 3 범위 밖** — 설계 §9 미결(ssrc 조회 경로/교차 시점). 별 토픽.

## §3 정지점 (2개)

- **★정지점 1 — 시각 기록 인프라**: 수신 RTP per-vssrc arrival 시각 + floor Granted/Idle 수신 시각이 한 타임라인에 모임.
- **★정지점 2 — 구간 gating 판정 + ptt_rapid 음성 검증 PASS**: 각 granted 구간에서 양성(그 봇 vssrc 도착) + 음성(타 봇 미도착, guard band 제외) 검산 통과.

## §4 단계별 작업

### Phase A — 봇 수신 시각 기록
- subscriber 수신 RTP에 per-vssrc arrival 시각 기록(구간 버킷 카운트로 충분, 패킷별 전체 보관 불필요). §0(나) 현재 카운팅 자리에 시각 축 추가.

### Phase B — floor 이벤트 시각 → ★정지점 1
- MBCP Granted/Idle 수신 시각을 phase 경계로 기록. 수신 RTP 타임라인과 합류.

### Phase C — 구간 분할 + gating 판정
- floor 경계로 구간 분할. 각 granted 구간: 그 봇 vssrc 도착(양성) + 타 봇 송신 미도착(음성).
- **guard band**: 전환 직후 구간 제외(silence flush/keyframe 잔류 덮기). 길이 상수화.

### Phase D — ptt_rapid 음성 검증 + 실측 → ★정지점 2
- ptt_rapid(회전) 실측에서 양성+음성 둘 다 PASS. 음성 위반(비발화자 누수) 시 어느 구간/vssrc인지 명시(서버 추적 단서).
