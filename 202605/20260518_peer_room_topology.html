§1개요

OxLens SFU 의 자료는 5층으로 구성:
  • 전역 (Global) — PeerMap (user-scope index 4종) / RoomHub (room-scope index)
  • user-scope — Peer (user 당 1개, cross-room 공유). PC pair 자원 소유.
  • PC-scope — PublishContext / SubscribeContext (PC 단위 자원 컨테이너)
  • room-scope — Room (방 당 1개) / RoomMember (방-user 멤버십)
  • stream-scope — PublisherStream / SubscriberStream / Slot (RTP 흐름 단위)
핵심 원칙: PC 자원 = Peer 소유 (Room 아님). RoomMember.peer: Arc<Peer> 로 user-scope 위임. cross-room 시 user 1명 = Peer 1개, 방마다 RoomMember 별도.

§3자료별 카드

PeerMap global
전역 user-scope index. 4-index DashMap 으로 user 1명을 4 경로 모두 O(1) 조회.
  • by_user : DashMap<String, Arc<Peer>>
  • by_ufrag : DashMap<String, (Arc<Peer>, PcType)>
  • by_addr : DashMap<SocketAddr, (Arc<Peer>, PcType)>
  • by_room_subscriber : DashMap<RoomId, DashMap<String, Arc<Peer>>>
RoomHub global
전역 room-scope index. 방 자료 소유 자리.
  • rooms : DashMap<RoomId, Arc<Room>>
Peer user-scope
user × sfud 단일 세션 컨테이너. PC pair 소유, cross-room 시 공유. 방 수와 무관하게 user 당 1개.
  • user_id : String
  • phase : AtomicU8 (Alive/Suspect/Zombie)
  • pub_room : ArcSwap<Option<RoomId>> · 1방 발언
  • sub_rooms : ArcSwap<RoomSet> · N방 청취
  • publish : PublishContext
  • subscribe : SubscribeContext
  • pub_stats : PublishPipelineStats
  • last_seen / suspect_since : AtomicU64
PublishContext PC-scope
Publish PC 자원 컨테이너. 클라이언트 → 서버 RTP 수신 자리.
  • media : MediaSession (ICE/DTLS/SRTP)
  • streams : ArcSwap<PublisherStreamIndex> · RCU
  • stream_map : Mutex<MediaIntent>
  • recv_stats : DashMap<u32, Mutex<RecvStats>>
  • twcc_recorder · pli_state
  • dc : DcState · DataChannel
  • rtx_seq · last_video_rtp_ms
SubscribeContext PC-scope
Subscribe PC 자원 컨테이너. 서버 → 클라이언트 RTP 송신 자리.
  • media : MediaSession (ICE/DTLS/SRTP)
  • streams : ArcSwap<SubscriberStreamIndex> · RCU
  • mid_pool : Mutex<MidPool> · mid 할당/재활용
  • mid_map : HashMap<track_id, (mid, kind)>
  • egress_tx/rx : mpsc<EgressPacket>
  • nack_suppress · rtx_budget
Room room-scope
방 자료. 방 멤버십 + Floor + PTT Slot 소유. PC 자원은 보유 X (Peer 가 소유).
  • id : RoomId
  • capacity · active_since · rec
  • floor : FloorController (PTT 발언권)
  • slots : Vec<Arc<Slot>> · [0] Audio / [1] Video
  • participants : DashMap<user_id, Arc<RoomMember>>
  • speaker_tracker : Mutex<SpeakerTracker> (RFC 6464 EWMA)
RoomMember room × user
방-user 멤버십. user-scope state 는 peer 에 위임. 방별 N개 존재.
  • room_id · role · joined_at
  • peer : Arc<Peer> · user-scope 위임
  • sub_stats : SubscribePipelineStats · room-scope
PublisherStream SSRC 단위
publisher 가 발행하는 RTP 흐름 1개. Janus videoroom 정합. Peer.publish.streams 가 소유.
  • ssrc : u32 · identity
  • user_id : Arc<str>
  • kind : TrackKind (Audio/Video)
  • duplex : AtomicU8 (Full/Half)
  • simulcast · video_codec · actual_pt
  • rtx_ssrc · rtp_cache · virtual_ssrc
  • phase : AtomicU8 (Created/Intended/Active)
SubscriberStream mid 단위
subscriber 가 수신하는 (publisher × kind) 단위 fan-out. LiveKit DownTrack 정합. Peer.subscribe.streams 가 소유.
  • mid : u8
  • virtual_ssrc : u32
  • kind · subscriber_id
  • publisher_ref : ArcSwap<PublisherRef> · Direct/ViaSlot/None
  • mode : SubscribeMode
  • room_stats : DashMap<RoomId, Arc<RoomStats>>
  • gate · pli_state · phase
Slot PTT virtual
PTT half-duplex 트랙용 가상 트랙. per-room random virtual_ssrc alloc. Room.slots 가 소유.
  • virtual_ssrc : u32 · 방마다 random
  • kind : SlotKind (Audio/Video)
  • room_id · rewriter (PttRewriter)

§4관계 자리 표

FromTo관계자리
PeerMap.by_user Peer owns Arc user_id → Arc<Peer>
RoomHub.rooms Room owns Arc RoomId → Arc<Room>
Peer.pub_room Room RoomId 참조 ArcSwap<Option<RoomId>> · 1방 발언
Peer.sub_rooms Room RoomSet 참조 ArcSwap<RoomSet> · N방 청취
Room.participants RoomMember owns Arc DashMap<user_id, Arc<RoomMember>>
RoomMember.peer Peer Arc 역참조 user-scope state 위임
Peer.publish PublishContext owns (값) PC pair · Pub 측
Peer.subscribe SubscribeContext owns (값) PC pair · Sub 측
PublishContext.streams PublisherStream owns Arc ArcSwap<PublisherStreamIndex> RCU
SubscribeContext.streams SubscriberStream owns Arc ArcSwap<SubscriberStreamIndex> RCU
Room.slots Slot owns Arc [0] Audio / [1] Video
SubscriberStream.publisher_ref PublisherStream Weak (Direct) full-duplex / simulcast · 1:1
SubscriberStream.publisher_ref Slot Weak (ViaSlot) half-duplex (PTT) · N:1
PublisherStream.peer_ref Peer Weak Peer drop 시 lazy upgrade None

§5핵심 원칙 (불변)

위반 시 버그 — 부장님 자세

  • PC 소유자 = Peer (Engine 자리) · Room 은 PC 자원 보유 X. Peer.publish / .subscribe 에 PC pair 자원 집중.
  • user-scope state 는 Peer 가 마스터 · RoomMember.peer: Arc<Peer> 로 위임. cross-room 시 user 1명 = Peer 1개, 방마다 RoomMember 별도.
  • pub_room 은 단수 (1방 발언) · ArcSwap<Option<RoomId>> 타입이 강제. sub_rooms 는 RoomSet (N방 청취).
  • subscribe mid 는 서버가 할당 · 클라 자체 할당 = m-line 무한 누적 버그. SubscribeContext.mid_pool 가 진실의 단일 원천.
  • PTT Slot 의 virtual_ssrc 는 per-room random · universal 상수 (PTT_AUDIO_SSRC) 폐기. cross-room affiliate 시 두 방이 동시 발언해도 NetEQ 충돌 없음 (Phase ①.5, 2026-05-02).
  • half-duplex → simulcast 강제 off · TrackType::HalfNonSim 단일 자리. PttRewriter + SimulcastRewriter 충돌 회피.
  • SubscriberStream.publisher_ref 는 Weak · Direct (1:1) / ViaSlot (N:1) / None (정리). publisher Peer drop 시 lazy upgrade None — strong cycle 자리 없음.
  • fan-out 핫패스는 lock-free · ArcSwap<PublisherStreamIndex>.load() ~4ns. MediaSession.is_media_ready() = atomic 단일 load.
