## §0 의무 점검
1. **서버 계약 확인처(읽기, 무수정)**: `oxlens-sfu-server/.../handler/track_ops.rs` `handle_mute_update` —
   - 요청 `{ track_id?, ssrc, muted }` (**track_id 우선, ssrc 하위호환**), 응답 `{ ssrc, muted }`.
   - `mute_stream(track_id, ssrc, muted)` = **Stream 단위**(simulcast h/l 동시). broadcast `TRACK_STATE { user_id, ssrc, track_id, kind, muted }`(본인 제외).

4. **core/ 무수정.** 대상 `sdk/domain/{local-pipe,local-endpoint,room,remote-pipe}.js` + `sdk/engine.js` + `_e2e/`.

## §1 컨텍스트 — 무엇이 있고 무엇이 없는가

서버는 **완비**다: MUTE_UPDATE 처리 + TRACK_STATE broadcast + VIDEO_SUSPENDED/RESUMED + unmute PLI. **클라만 미정합.**

- **송신측 없음**: LocalPipe 에 mute 토글이 없다. `track.enabled=false` + MUTE_UPDATE 송신 경로 신설.
- **수신측 부분**: signaling 이 `track:state` emit(확인 — 3b signaling). 하지만 **Room 이 `track:state` 구독 안 함**(3e 가 tracks:update/room:event 만 구독). muted 필드 → RemotePipe avatar 전환 경로 없음. VIDEO_SUSPENDED/RESUMED 도 미구독.

**완료 정의**: full-duplex 참가자가 mic/camera mute → `track.enabled=false` + MUTE_UPDATE(track_id 우선) → 상대가 TRACK_STATE 수신 → avatar/placeholder 전환. unmute 역. **라이브 2-peer 검증**(alice mute → bob 화면 avatar, alice unmute → 복원 + 서버 PLI 로 키프레임).

## §2 결정된 사항 (서버 계약 + 설계 §13 정합)

- **mute = `track.enabled` 토글** (replaceTrack 금지 — SSRC/BWE/transceiver 보존, 마스터 원칙). LocalPipe 가 게이트.
- **track_id 우선 송신**(§13.4): MUTE_UPDATE body `{ track_id, ssrc, muted }`. track_id = 서버 학습값(3b `_learnTrackId`). ssrc 는 하위호환 동봉(서버가 track_id 우선).
- **Stream 단위**(서버 `mute_stream`): 클라는 source(mic/camera) 단위 LocalPipe 1개 = track_id 1개 → 그대로 송신. simulcast h/l 은 서버가 Stream 단위로 동시 처리(클라 무관).
- **수신 = TRACK_STATE 구독**: Room 이 `track:state` bus 구독 → `muted` 필드 → 해당 RemotePipe avatar 전환(`media:track:state` 또는 기존 `media:track` 확장 emit). VIDEO_SUSPENDED/RESUMED 도 구독(중복 신호 — muted 와 정합, avatar 일관).
- **PTT 제외**: half-duplex 트랙은 mute(enabled) 안 함(floor gating 이 담당). 3c = full-duplex(mic full / camera full)만.
- **범위 밖**: half/PTT mute / Power(HOT/COLD) / 카메라 디바이스 on-off(replaceTrack) 는 §3 판단 후 별 / observability / TransportSet.

## §4 단계별 작업 (Phase 3c = A~E)

### Phase A — LocalPipe.setMuted (★ 정지점 1)
`sdk/domain/local-pipe.js`:
- `setMuted(bool)`: `this.track.enabled = !bool` (track 있을 때). `this._muted = bool` 상태 보유. **replaceTrack 금지**(SSRC/transceiver 보존). track 없으면(미publish) no-op + 상태만.
- getter `get muted()`.
- trackState 와 독립 — mute 는 ACTIVE 유지(트랙 살아있음, 전송만 끔). RELEASED/INACTIVE 와 혼동 금지.

### Phase B — LocalEndpoint mute + MUTE_UPDATE 송신 (★ 정지점 1)
`sdk/domain/local-endpoint.js`:
- `setMuted(source, bool)`: `getPipeBySource(source)` → `pipe.setMuted(bool)` → MUTE_UPDATE 송신.
  - body: `{ track_id: pipe.trackId, ssrc: pipe.ssrc ?? 0, muted: bool }`. **track_id 우선**(서버 학습값). publish 안 된(trackId=null) source 면 enabled 만 토글 + 송신 skip + 경고.
  - **send vs request**: MUTE_UPDATE 응답 `{ssrc,muted}` 단순 ack — `sig.send` 충분(await 불요) or request(확인용). 판단 후 보고. (3b 결함처럼 응답 본문 쓸 일 없으면 send.)
- full-duplex 가드: `pipe.duplex === 'half'` 면 mute 금지(PTT — floor gating). 경고 + skip.

### Phase C — engine 공개 API + Room TRACK_STATE 구독 (★ 정지점 1)
`sdk/engine.js`:
- `setMuted(source, bool)` → `localEndpoint.setMuted`. + facade(`muteMic/unmuteMic/muteCamera/unmuteCamera`) — §3 형태 판단.
`sdk/domain/room.js`:
- 생성자 `bus.on('track:state', ...)` → `_onTrackState(d)`: room_id 필터 → `{user_id, track_id, kind, muted}` → 해당 RemotePipe(matchPipeByTrackId or userId+kind) → `pipe.setRemoteMuted(muted)` → `media:track:state` emit(앱이 avatar 전환).
- `bus.on('video:suspended'/'video:resumed', ...)`: 보조 — muted 와 정합(중복 시 무시 or avatar 일관). §3 권위 판단.
- teardown 에 off 추가(3e 패턴).

### Phase D — RemotePipe avatar (★ 정지점 1)
`sdk/domain/remote-pipe.js`:
- `setRemoteMuted(bool)`: video element 숨김/placeholder(mute=일시, 3e unmount=제거와 구분). `_remoteMuted` 상태.
- audio mute: track.enabled 영향 없음(수신측은 서버가 안 보냄 — muted 면 RTP 자체 감소). UI 표시만.

### Phase E — mock + 라이브 (★ 정지점 1 mock / ★ 정지점 2 라이브)
- `sdk/_t3c_check.mjs`: setMuted → track.enabled=false + MUTE_UPDATE body(track_id 우선) 확인 / track:state 수신 → RemotePipe avatar / 회귀(_t3e/_t3d/_t3b2 PASS).
- 라이브(harness 확장 `?scenario=mute`): alice publish → mute mic+camera → bob 화면 avatar + TRACK_STATE 수신 / alice unmute → 복원 + 서버 PLI 로 video 키프레임 도착(admin 교차: publish rtp_in 흐름 / TRACK_STATE muted broadcast).
- 라이브 차단 시 Phase G 교훈: 자명한 키/스키마 mismatch 즉석 패치+보고, 설계 판단 멈춤.

## §6 기각된 접근법
- **mute = replaceTrack(null)** — SSRC/BWE/transceiver 파괴. `track.enabled` 토글이 정답(마스터 원칙). replaceTrack(null) 은 "디바이스 off" 별 의미.
- **PTT half mute(enabled 토글)** — floor gating 이 담당, audio enabled 항상 true(마스터). 3c full-duplex 만.
- **ssrc 단독 송신(track_id 무시)** — 서버 track_id 우선(§13.4). track_id 동봉.
- **클라가 simulcast h/l 개별 mute** — 서버 mute_stream 이 Stream 단위 동시. 클라는 source 1개 송신.
- **unmute 시 클라 PLI 송신** — 서버가 보냄(handle_mute_update). 클라 불요.
- **VIDEO_SUSPENDED 를 mute 권위로** — TRACK_STATE muted 가 권위(track_id 동반). VIDEO_SUSPENDED 보조.
