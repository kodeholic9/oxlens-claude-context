### Phase B — 서버 파일 0602e 현행화 (★정지점)
`PROJECT_SERVER.md` 의 stale 항목을 아래 **정답 사실**로 교체. domain/ 트리는 **실제 디렉토리(`ls crates/oxsfud/src/domain/`) 기준**으로 김과장이 갱신(mechanical). 아키텍처 서술은 아래 사실대로 + 코드 교차:

- **(B-1) 서버 소스 구조**: `room/` → **`domain/`** 전면 교체. 신규 파일 반영: `types.rs`(enum 단일출처)·`publisher_track_index.rs`·`subscriber_stream_index.rs`·`floor_routing.rs`·`peer_map.rs`·`state.rs`(3-Layer State). 폐기 반영: `stream_map.rs`·`scope.rs` 제거.
- **(B-2) enum 단일출처**: `StreamKind`/`TrackKind`/`VideoCodec`/`DuplexMode`/`TrackType` = `domain/types.rs` (구 stream_map/participant/publisher_track 분산 → 통합).
- **(B-3) fan-out 방향 역전** (0528~0601): `PublisherTrack.subscribers: ArcSwap<Vec<Weak<SubscriberStream>>>` 직접 보유. `fanout() → track_type() → broadcast()` 단일 본문 — Full=`self.subscribers` / Half=방 `Slot.subscribers` 직접 순회. **vssrc 역탐색(`find_subscriber_stream_by_vssrc`) 폐기**. (구 "vssrc 매칭 fan-out" 서술 전면 교체.)
- **(B-4) stats 트랙 차원 정렬** (0602b~c): `RecvStats→RrStats` / `SendStats→SrStats` 개명. `PublishPipelineStats` = **PublisherTrack 직속**(구 PublishContext.pub_stats). `SubscribePipelineStats` = **subscriber_stream 직속**(구 RoomMember.sub_stats). `room_stats` DashMap 제거 + `room_id` 정체 필드 승격. admin 은 트랙/room 합산.
- **(B-5) scope 자료구조** (0602e): `RoomSet`/`RoomSetId` 래퍼 폐기 → `sub_rooms: ArcSwap<HashSet<RoomId>>`. `set_id`(sub_set_id/pub_set_id) wire 필드 폐기. 다방 청취·SCOPE op 유지.
- **(B-6) Peer 재설계 자료 위치**: publish 자료 home 갱신 — `pub_stats` 가 PublishContext → PublisherTrack 직속 이동(B-4). `pli_state` 는 논리 PublisherStream(2계층).
- **(B-7) 2계층**: "Stage 2~4 미커밋" → **전부 커밋 완료**. (`PublishContext ⊃ PublisherStream(논리) ⊃ PublisherTrack(물리)` 확정.)

### Phase C — track_state 식별 계층 반영 (`PROJECT_SERVER.md`)
이번 서버 작업(8b0627a/208a498 + track_id 통지 정정) 반영:
- **식별 3평면**: `track_id`(불투명, 논리 `PublisherStream` 소유, 파싱 금지·키 조회) / `vssrc`(논리 Stream, simulcast egress 원천, **라우팅 키 아님** — subscribe 등록 cold-path 에서 SubscriberStream.virtual_ssrc 로 복사) / 실 `ssrc`(물리 PublisherTrack).
- track_id 생성 `{user}_{대표 ssrc}`(non-sim/PTT=원본ssrc, simulcast=vssrc). `attach_track_to_stream` 발급.
- `find_stream_by_track_id`/`find_stream_by_vssrc`(vssrc=0 가드) 키 조회. `switch_track_duplex` 폐기(dead 청산). mute = Stream 단위(h/l 비대칭 해소). simulcast duplex reject.
