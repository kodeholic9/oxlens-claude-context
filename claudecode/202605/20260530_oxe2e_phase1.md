## §0 의무 점검 (시작 전 반드시 — 대량 읽기는 여기서 김과장이 직접)

아래 파일을 읽고 **대조**한다. 김대리는 골격(무엇을)만 준다. **정확한 oxrtc API 시그니처(어느 함수/타입)는 김과장이 이 대조로 직접 도출**한다.

**(가) 설계 근거 — 먼저**
- `context/design/20260530_oxe2e_design.md` — 이 작업의 전체 근거. 특히 §2(계승/폐기/신규), §6(판정 모델), §7(스키마).

**(나) oxrtc 봇 진입 API — 핵심 대조 대상**
- `oxlens-sfu-server/crates/oxrtc/src/lib.rs` — 공개 표면. 봇이 쓸 수 있는 게 뭔지.
- `oxrtc/src/signal_client.rs` — v3 WS 시그널링 클라이언트. IDENTIFY/JOIN/PUBLISH_TRACKS/TRACKS_READY 송신 + 이벤트 수신 API.
- `oxrtc/src/ice_client.rs` / `dtls_client.rs` / `srtp_session.rs` — transport 셋업.
- `oxrtc/src/rtp_receiver.rs` / `rtcp_sender.rs` — RTP 수신·RTCP 송신.
- `oxrtc/src/error.rs` — LiveResult 등 에러 타입.

**(다) labs 봇 현 구조 — 패턴 참조용 (이주 아님)**
- `oxlens-sfu-labs/crates/oxlab-bot/src/` 전체 — 봇이 *무엇을 하는지*(시그널링 흐름, Fake RTP 송수신, PLI 응답 패턴)를 본다.
- **주의: labs 봇은 v2 시그널링(opcode 십진수, TRACKS_ACK=16 등)에 멈춰 있다. 코드를 그대로 가져오지 말 것.** 흐름만 참고하고 oxrtc(v3)로 새로 짠다.

**(라) 서버가 "약속"을 어떻게 보내는지 — 판정 근거**

- `PROJECT_MASTER.md` 시그널링 v3 섹션 + `context/design/wire_v3_catalog.md` — opcode 카탈로그.

**대조 산출**: "labs 봇의 이 단계 → oxrtc 의 이 API" 매핑표를 머릿속(또는 메모)으로 작성. 이게 Phase B~D 구현의 기반.

## §1 컨텍스트

- **oxe2e** = 단위 시험(cargo test, 미디어 흐름 없음) ↔ 브라우저 E2E(무겁다) 사이의 헤드리스 회귀 gate. 리팩토링 후 "뭐 깨졌나"를 김과장이 `oxe2e scenario x.toml` 한 줄, 3~5분에 PASS/FAIL로 확인.
- **Phase 1 목표**: 봇이 oxrtc(v3)로 서버에 붙어 **conf_basic**(전환 없는 conference)을 돌리고, **약속↔이행 판정**으로 PASS/FAIL을 낸다.
- **왜 oxrtc 전환부터**: labs 봇은 v2 자체구현이라 v3 서버와 wire가 안 맞는다. oxrtc는 이미 v3 signal_client 완성 → 봇이 oxrtc를 쓰면 wire가 자동 정합한다. 가장 싼 부활 경로.
- **floor / DC / SCTP 는 Phase 2** (별도 지침). 이번엔 손대지 않는다.

## §2 결정된 사항 (설계 확정 — 변경 금지)

1. **위치**: `oxlens-sfu-server/crates/oxe2e/` 신설. workspace member 추가.
2. **참조**: 같은 workspace **path dependency** — `oxrtc` + `oxsig`. (git dep 아님.)
3. **봇 = 새 작성.** labs 봇 코드 이주 아님. 흐름만 참조, oxrtc(v3) 기반 새로.
4. **판정 = 약속↔이행 범용.** 기능별 체크포인트(L1-XX) 만들지 않는다.
   - 약속 = TRACKS_UPDATE(add)의 ssrc/mid. 이행 = 그 ssrc가 RTP로 도착했나.
   - 존재 = 조합 당위(N명 full → 각자 N-1명 수신)상 add가 다 왔나.
5. **충실도 경계 (conf_basic)**: 서버가 읽는 것만 충실 — RTP 헤더(SSRC/seq/ts/marker/PT) + **mid extension**. 페이로드 본문은 더미. (conf_basic은 전환이 없어 keyframe descriptor 충실도까지는 불필요. simulcast/PTT 갈 때 보강.)
6. **admin 삼각검증은 Phase 1 범위 밖** — 봇 단독 약속↔이행부터. admin 교차(track-dump 4-Point)는 conf_basic이 선 다음 후속.
7. **서버 코드 0 변경.** oxe2e는 oxrtc/oxsig를 소비만 한다.

## §3 결정 추천 (★ 정지점 2개)

- **★ 정지점 1 — 시그널링 핸드셰이크 성공**
  oxe2e crate 골격 + 봇이 oxrtc로 WS 연결 → IDENTIFY → ROOM_JOIN → server_config 수신까지 성공.
  - 이유: oxrtc 봇 진입 API가 맞는지, wire가 실제 정합하는지가 첫 리스크. 시그널링이 붙어야 그 위가 의미. 여기서 **commit + 보고 + GO 대기**.
- **★ 정지점 2 — conf_basic 완성**
  봇 N개 publish + RTP 송수신 + 약속↔이행 판정 PASS/FAIL 출력. `oxe2e scenario conf_basic.toml` 동작.
  - 여기서 **commit + 보고**.

그 외 단계는 정지점 없이 통합 진행.

## §4 단계별 작업

### Phase A — oxe2e crate 골격
- `crates/oxe2e/` + `Cargo.toml` (workspace member 등록, path dep `oxrtc`/`oxsig`).
- `src/main.rs` — CLI 진입 `oxe2e scenario <file>` (인자 파싱 최소).
- `src/lib.rs` + 모듈 골격: `bot/`, `scenario/`, `judge/`. (설계서 §6/§7 참조하되 새로.)
- `cargo check` 통과.

### Phase B — 봇 시그널링 (oxrtc signal_client) → ★정지점 1
- oxrtc signal_client로: WS 연결 → IDENTIFY → ROOM_JOIN → `server_config` 수신.
- 정확한 메서드/타입은 §0(나) 대조로 확정.
- 봇 1개로 핸드셰이크만 먼저 검증 → **commit + 보고 + GO**.

### Phase C — 봇 transport (oxrtc ICE/DTLS/SRTP)
- localhost + 서버 **ICE-Lite** → 봇은 **STUN binding만**. full-ICE 스택 짜지 말 것.
- pub PC: DTLS + SRTP 셋업. sub PC: 수신 셋업.
- oxrtc가 이 셋업을 어디까지 캡슐화하는지 §0 대조로 확인. (이미 제공하면 호출만.)

### Phase D — 봇 RTP 송수신 (충실도 경계 §2-5)
- **Publisher**: Fake VP8 RTP(33ms 간격) + Fake Opus RTP(20ms). RTP 헤더 + mid extension 충실, 페이로드 더미.
- PLI 수신 → keyframe 플래그(다음 패킷에 marker 등 서버가 읽는 표식). conf_basic 범위에선 최소.
- TRACKS_UPDATE 수신 → **TRACKS_READY** 응답 (SubscriberGate 해소).
- **Subscriber**: RTP 수신 + per-SSRC 기록(어떤 ssrc로 몇 패킷 왔나). seq/ssrc 추적.

### Phase E — 약속↔이행 판정 (judge)
- **약속 수집**: TRACKS_UPDATE(add) 수신 시 ssrc/mid/owner 기록.
- **이행 검산**: 약속된 ssrc가 Subscriber RTP에 도착했나.
- **존재 검산**: conf_basic 조합 당위(전원 full audio+video → 각 봇이 나머지 전원 트랙 수신)상 add가 다 왔나.
- 결과: PASS/FAIL 텍스트 + exit code(0/1). 어긋난 항목은 "약속 X 미이행" 식으로 명시(서버 로그 추적 단서).

### Phase F — conf_basic 시나리오 + 실행 → ★정지점 2
- `scenarios/conf_basic.toml` — 참여자 2~3명(full audio+video), 타임라인 join→publish→wait(수 초).
- `oxe2e scenario conf_basic.toml` → PASS 확인.
- **commit + 보고**.

## §7 기각된 접근법 (유혹 차단)

- **labs 봇 v2 코드 그대로 이주** — wire 안 맞음. oxrtc(v3) 기반 새로.
- **봇 full-ICE 스택** — localhost + 서버 ICE-Lite라 STUN binding만이면 충분.
- **기능별 체크포인트(L1-01~21)** — 약속↔이행 범용 판정으로 대체. 봇이 못 잡는 미디어 품질(silence/keyframe-first)을 binary로 넣지 않는다.
- **실제 미디어 인코딩 전송** — conf_basic은 더미 페이로드. 실미디어는 labs 몫.
- **DC / SCTP / floor / MBCP** — Phase 2 별도 지침. 이번엔 손대지 말 것.
- **admin 삼각검증** — Phase 1은 봇 단독부터. conf_basic 선 다음 후속.
- **시나리오 자동생성** — 조합 수동 선언 TOML.
