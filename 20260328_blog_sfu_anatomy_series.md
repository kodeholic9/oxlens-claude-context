## 주요 수정 사항 및 학습

### 2편 수정
- 공유기 vs NAT 용어 구분 확정: 물리 장비/배치 → "공유기", 동작/정책 → "NAT"
- 포트포워딩 섹션 추가: SFU가 반드시 공인 IP를 직접 가질 필요 없음. 공유기 뒤 RPi + 포트포워딩으로 동작 가능. 핵심은 "클라이언트가 도달할 수 있는 고정된 공인 주소"

### 3편 수정 (2건)
1. **SRTP 헤더 평문 이유**: "SFU가 패킷을 열어봐야 하니 SFU 의미 없음"(틀림) → "복호화 전에 라우팅 판단이 먼저 필요. 헤더가 암호화되면 어떤 키로 복호화할지 모르는 닭과 달걀 문제"(정확)
2. **Auth Tag 조작 불가 단정**: "조작할 수 없다"(틀림) → "SRTP 키를 모르는 제3자는 조작할 수 없다. SFU처럼 키를 가진 종단점은 헤더 변경 + Auth Tag 재생성 가능"(정확). OxLens의 PttRewriter/SimulcastRewriter가 실제로 rewrite 수행 중

### 5편 수정
- 용어 벽 문제: 초/중급 대상인데 demux, SSRC, RTCP, payload type, fan-out 등 설명 없이 등장
- **해결 패턴: "행동 먼저, 이름 뒤에"** — 정의를 내리면 교과서가 됨. 뭘 하는지 먼저 보여주고 이름을 뒤에 붙이는 방식으로 자연스럽게 녹임
- fan-out → "복사 전달"로 치환 (용어 자체를 안 씀)
