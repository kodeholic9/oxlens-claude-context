## §0 요약

- **filter 부착 위치 = 인코더 "앞"** 결론(업계 3사 조사). encoded transform(인코더 뒤)은 E2EE 전용 — filter로 쓰는 곳 없음.
- sdk0.2 접점 = **`LocalPipe.setProcessor`**(track→track→`swapTrack` 단일 게이트, SSRC 보존). 이름 = **Processor**(기존 `setProcessor`·LiveKit 정합).
- 구현: 인터페이스(`AudioProcessor`/`VideoProcessor`) + 파이프라인(Web Audio / Canvas, v2 이식) + 내장 3종(RadioVoice/NoiseSuppress/BackgroundBlur).
- 검증: `tsc` 0에러 · `tsup` 빌드 성공. 3층 라이브 `conf_processor` **2/2**(radioVoice FFT 43dB 상대부스트 / blur Canvas 640×480 live). 전체 회귀 **6/6**.

## §1 조사 (부장님 지시: LiveKit/미숩/집시 + 웹)

- **reference/**: livekit(Go SFU 서버)·janus(C 서버)·mediasoup-client·**lib-jitsi-meet** — 클라 filter 프레임워크는 Jitsi/Live-track-processors만.
- **결론 3사 만장일치**:
  | | filter(픽셀/샘플) | encoded transform |
  |---|---|---|
  | LiveKit | `TrackProcessor`(video=MediaStreamTrackProcessor / audio=Web Audio) — 인코더 앞 | E2EE |
  | Jitsi | `setEffect`(`IStreamEffect.startEffect(stream)→stream`) — 인코더 앞 | **E2EE + litemode 전용** |
  | mediasoup | 없음(앱 가공 + `replaceTrack`) | (앱) |
  | 우리 v2 | Canvas + Web Audio — 인코더 앞 | 없음 |
- **encoded transform(RTCRtpScriptTransform) = 인코더 뒤, 이미 인코딩된 프레임 → 픽셀 가공 불가 → E2EE/메타 전용.** lib-jitsi-meet 의 insertable 사용처가 전부 `e2ee/*`·`LiteModeContext` 로 실증.
- **E2EE 는 현재 해당 없음**(SFU rewrite=PTT/simulcast 와 충돌 → 별 과제). filter 는 인코더 앞이라 무충돌.

## §2 설계 (`sdk0.2/design/processor-design.md`)

- **DX 최우선**: `const mic = await engine.localEndpoint.enableMicrophone(); mic.setProcessor(new RadioVoiceProcessor())`. 단일/배열(체인). `clearProcessor()` 원복.
- **접점 = 인코더 앞**: `swapTrack`(제2조 단일 게이트)로 원본↔가공 교체, 원본 보관→복원. 코어 침습 0(v2의 `_currentRoom`/`sender` 직접접근 폐기).
- **봉인**: 인터페이스·내장 Processor = 공개 export / 파이프라인 엔진 = `internal/media/processor/` 봉인.
- **인터페이스**: `AudioProcessor{setup(ctx,input)→node}` / `VideoProcessor{init?, process(ctx,frame), destroy?}`(v2 차용).

## §3 구현

- **신규 6파일**: `core/processor.ts`(인터페이스) · `internal/media/processor/{audio,video}-pipeline.ts`(v2 media-filter.js 이식) · `processors/{radio-voice,noise-suppress,background-blur}.ts`(v2 filters 이식).
- **`LocalPipe`**: `setProcessor(p|p[])`/`clearProcessor()`/`hasProcessor` + `_disposeProcessing()`(release/deactivate/teardown 종료경로 훅, 원본 track 정리). kind 로 audio/video 파이프라인 분기.
- **`LocalEndpoint`**: 기존 1회성 `setProcessor(source,processor)`/`stopProcessor` → **pipe 위임**(`setProcessor`/`clearProcessor(source)`)로 통일. 죽은 `_processed` Map 제거.
- **audio 파이프라인**: `track → MediaStreamAudioSource → [processor.setup 체인] → MediaStreamAudioDestination → track`(Web Audio, Safari 호환).
- **video 파이프라인**: `track → hidden<video> → Canvas(drawImage) → [processor.process] → captureStream → track`(rAF, 정적화면 대응). **우선 Canvas**(insertable 최적화는 후속).
- **내장**: RadioVoice(bandpass1500+comp+gain) / NoiseSuppress(hp80+lp8k+comp, placeholder) / BackgroundBlur(canvas blur+중앙타원, placeholder).
- `index.ts` export + `qa/qa.js` `window.qa.processor`(set/clear/has/track/spectrum) 표면.

## §4 시험 (3층 라이브 — `qa/live/tests/conf_processor.spec.ts`)

- **PROC-01 mic radioVoice**: 원본 스펙트럼(FFT) 캡처 → setProcessor → `applied`·`hasProcessor`·track 교체 → 가공 스펙트럼 → **clear 복원**.
  - gain+comp 로 절대레벨 +90dB 상승 → 절대비교 무의미. **상대 형태**로 bandpass 증명: 통과대역(1500)이 원본 대비 저주파(200) 상대로 **+43dB 부스트**(orig -53.5dB → proc -10.3dB). 고주파(5000) 상대 감쇠(-12.3→-16.0dB). 재현성 확인(2회 동일).
- **PROC-02 camera blur**: setProcessor(blur)→`applied`·Canvas track 교체(원본과 다름)·640×480 live·**clear 복원**. video Canvas 경로 실증.
- **전체 회귀 6/6**: duplex/multiroom/republish/simulcast/video + processor. LocalPipe/LocalEndpoint 수정에도 회귀 없음.
