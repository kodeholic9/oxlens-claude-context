## §1 설계 결정 (부장님)

- **진입점 = `Engine`**(어휘 계승). 공개 path 2개뿐: `engine.join`(단일방 컨퍼런스) / `engine.talkgroups`(다방, `.talk`=발화권). 복합 시나리오는 talkgroups가 흡수(프리셋 12종 매핑).
- **어휘 계승**: `Engine`/`Room`/`LocalEndpoint`/`RemoteEndpoint`/`LocalPipe`/`RemotePipe`/`floor`/`power` — v0.6 그대로. 바뀐 것은 구조(2-path)·주석 체계·TS 전환뿐. (participant→Endpoint, track→Pipe 정정: 부장님 "기존것 이름 괜찮다".)
- **internal 봉인**: `wire`/`transport`/`sdp`/`mbcp`/`dc-frame` → `internal/`(index 배럴 미노출).
- **Stream 래퍼 폐기**: `RemotePipe`가 곧 개발자 핸들(`attach(el)`/`setQuality`).
- **IoC 훅 릴레이 금지(부장님 핵심 지적 — 난독 원인)**: `Room.setRenegoHook`/`setLeaveHook`, talkgroups `assembleRoom`/`destroyRoom`/`migratePubAxis`(fn 주입), ptt `onMediaTrack`/`getRoom` 콜백 전부 제거 → engine 직접 호출·직접 소유(`RoomHost` 인터페이스). 부수효과 오케스트레이션은 engine이 관장. 계층 경계 사실 콜백(signaling/transport→engine)만 1단 유지(릴레이 아님).
- **느슨한 TS**: `strict:false`·`noImplicitAny:false`·`strictNullChecks:true`만. 공개 표면만 견고. tsup 배포, 런타임 무의존.
- **관측 계층 보류**: telemetry/event-reporter/user-probe는 **v0.3**. observe는 `diagnostics` 훅으로 최소화.

## §3 cross-sfu PTT slot GAP 해소

- **증상**: 다방(policy=multi) 청취방(affiliate, cross-sfu = 다른 SFU)의 slot 미디어 미수신(RB slot Δ=-1). home(RA)만 수신.
- **진단(부장님 통찰)**: `virtual._slots`(방별 Map) + `_byMid`(전역 mid 역인덱스)는 `rooms` Map의 **중복 자료구조**. `_byMid`가 mid만 키라 cross-sfu에서 mid 충돌(mid는 SFU 로컬 — 제4조).
- **수정(최소 침습, 복잡도 감소)**: `_byMid` **폐기** → `slotByMid(roomId, mid)`. `engine._onTrackReceived`를 sfuId 라우팅 **단일 경로**로 통합(일반 track과 동일하게 `sfuId` 가드 후 매칭). mid가 Room(=한 SFU) 안에서 유일하므로 충돌 소멸.
- **뿌리 2개**: ① SDK `_byMid`(위) ② `qa.js` `affiliate`가 청취방을 `wireRoom` 안 해 `trackSubscribed` 미구독(**하네스 누락 — SDK 아님**. v0.1에서 "SDK GAP"으로 오인·박제됐던 정체 가능성).
- **결과**: RB slot Δ=125~126, 양방(home+listen) 수신. `conf_multiroom` GAP 관측 → **실단언 승격**(`expect(RB Δ).toBeGreaterThan(0)`).

## §4 시험 (3층 라이브)

- `qa/qa.js`를 **v0.2 어댑터로 재작성**: `window.qa.*` 표면(및 `conn:'identified'` 계약)을 v0.1 그대로 유지 → `fixtures/participant.ts` + spec 5종 **무변경 재사용**. 내부만 v0.2 배선(`engine.join`/`localEndpoint.enable*`/`talkgroups.talk`/`trackSubscribed(pipe,endpoint)`/source `mic`→`microphone` 매핑).
- 라이브 회귀 **5/5**: conf_video(3인 3권위 교차, packetsReceived Δ+oxadmin passthrough) / republish / simulcast(h→l) / duplex(half↔full) / multiroom(양방 slot).
