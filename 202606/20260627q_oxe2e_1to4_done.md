## 칸 1 — floor 손바뀜 seam ✅

**선행 측정**(`ptt_voice_seam.yaml`, release→request 0.5s→0.1s): 무음갭 = 의도간격 + 서버상수 분리.
- 0.5s 간격 → 무음갭 0.498s, 0.1s 간격 → 무음갭 0.103s. **서버 순수 전환 = 잔류~3ms / priming~1ms**(간격 무관 상수 확증).
- **slot seq 손바뀜 +1 연속**(seq 143→144, 97→98) — ptt_rewriter SSRC/seq/ts 재매핑 실측.

**등식 `floor_seam`** (gating_correct 가 ±250ms guard 로 면제하는 구간을 정면):
- (a) RELEASE→옛화자 잔류·GRANTED→새화자 priming ≤ 50ms(의도 무음 제외, 서버 비용만).
- (b) 경계 slot seq 단조(+1~5). (c) ±guard 제3자 audio 유입 0.
- 음성 픽스처 6종. 라이브 ptt_voice 위반 0.

## 칸 2 — S3 누설종단 ⛔ 막힘

선행 조사: **봇이 RTCP 미송신**(`transport.py` send_rtp 만, aiortc receiver 미사용 raw intercept).
- subscriber RR 미송신 → RR 누설검사 불가. publisher SR 미송신 → SR translate 검사 불가.
- → `GAP-S3-rtcp-terminate` 등록. covered_by: 봇 RR/SR 송신 + loader RTCP 내용 파싱.

## 칸 3 — L4 복구발화 ⛔ 막힘

선행 조사(conf_audio_fault 실측): fault drop 으로 seq 결손 생성되나 **서버→봇 RTCP = RR 만, NACK 0, RTX(PT=97) 0**.
- subscriber 봇 NACK 미송신(S3 동근) → 서버 NackGenerator 무트리거 → RTX 미발생.
- → `GAP-L4-recovery-signal` 등록. covered_by: 봇 NACK 송신 + loader NACK/RTX 적재(S3 토대 공유).

## 칸 4 — L1 경합 ✅(경합부) / 급사 GAP

**선행 조사**: 봇이 모든 floor 메시지(DENY/TAKEN/REVOKE) record(bot.py:238). 동시 REQUEST **결정적**(botA 2회 동일 GRANTED).

**등식 `floor_convergence`**: 동시 REQUEST(±100ms 클러스터) → GRANTED 정확히 1(2명=중재깨짐 FAIL, 0명=교착 FAIL).
- 라이브 `conf_floor_contention`: 경합(botA win) + **queue 자동 승계**(A release@3.52 → B grant@3.52) 실측. 위반 0.
- 음성 픽스처 4종(정상/2명grant/교착/skip).
- **holder 급사 회수**(중도 DC 끊김→reaper 회수)는 봇 disconnect 액션 + reaper 시간 의존 → `GAP-L1-floor-failover` 분리.
