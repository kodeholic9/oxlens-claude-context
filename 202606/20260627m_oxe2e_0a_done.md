- under-fanout: 받아야 할 다른 publisher 누락(client_pub 전수 기준 → identity 보다 상위, server_sub add 누락도 적발).

## 3. 갈래B(failability) — 일회성 합성에서 회귀 스위트로 승격

**workplan §1 의 갈래B 표는 일회성 합성 실증으로 끝나 tests/ 에 영구 픽스처가 없었다**(가이드 §3 "등식마다 음성 픽스처 짝 의무 — 안 깨지는 등식은 게이트 아님" 미충족). 커밋 전 적발 → 4종 등록:

| 픽스처 | 기대 |
|---|---|
| `test_fanout_complete_pass` | 정상 N=3 → PASS |
| `test_fanout_complete_self_echo_fail` | self-echo 주입 → FAIL |
| `test_fanout_complete_under_fanout_fail` | under-fanout 주입 → FAIL |
| `test_fanout_complete_skip_under_3` | N=2 → 공허 skip |

self-echo/under-fanout 픽스처는 `assert any(... in v)` 구조라 등식 무력화(`[]` 반환) 시 곧바로 빨강 = 게이트 작동.
